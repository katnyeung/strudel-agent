How It Works

EXTRACT: Query evolution_events joined with sessions (for skill_id) since last watermark. Also fetch ratings in the same window.

TRANSFORM: For each event: extract voice names from code, compute play_secs (gap to next event), find closest rating within ±5 min.

LOAD: Write VoiceState nodes (voice combos with running averages) to Neo4j. Write TRANSITION edges between states where the voice set actually changed (no self-loops). Each edge tracks success rate and rating improvement.

After analysis: The agent queries: (1) transitions from current state — "what voice-changing move worked well from here?" and (2) array containment — "what voice should I add/remove to reach a high-rated state?"
